# 5. Lists and Keys

## Concept

- Rendering lists in React is done using `.map()`.
- Each list item needs a unique `key` prop for efficient updates.

## Why it matters in interviews

- You’ll often need to render dynamic lists (e.g., todos, users).
- Using keys correctly prevents bugs and improves performance.

## Example

```jsx
const fruits = ['Apple', 'Banana', 'Cherry'];

function FruitList() {
  return (
    <ul>
      {fruits.map(fruit => (
        <li key={fruit}>{fruit}</li>
      ))}
    </ul>
  );
}
```

## Key Points

- Keys should be unique and stable (avoid using array index if possible).
- Keys help React identify which items changed, added, or removed.

## Interview Questions & Answers

### Q1: Why do we need keys in React lists?

**Answer:** Keys help React identify which items have changed, been added, or been removed. Without keys (or using index as key), React uses position-based reconciliation, which can cause:
- Bugs when items are reordered or filtered
- Form input values getting mixed up
- Performance issues

**Example Problem Without Keys:**
```jsx
// ❌ BAD - Using index as key
function TodoList({ todos }) {
  return (
    <ul>
      {todos.map((todo, index) => (
        <li key={index}>{todo.text}</li>
      ))}
    </ul>
  );
}

// If you reorder or filter todos, the wrong one gets deleted
// Component state (like focused input) follows the wrong item
```

**Correct Solution:**
```jsx
// ✅ GOOD - Using unique id
function TodoList({ todos }) {
  return (
    <ul>
      {todos.map(todo => (
        <li key={todo.id}>{todo.text}</li>
      ))}
    </ul>
  );
}
```

---

### Q2: Can we use array index as a key?

**Answer:** Generally **NO**, except in these specific cases:
- List is **static** (no filtering, sorting, or reordering)
- List **never gets filtered**
- List has **no IDs**
- List is **never reordered**

**Real-world Problem with Index Keys:**
```jsx
const [items, setItems] = useState([
  { id: 1, name: 'Item 1', checked: false },
  { id: 2, name: 'Item 2', checked: false }
]);

function ItemList() {
  return (
    <ul>
      {items.map((item, index) => (
        <li key={index}>
          <input type="checkbox" />
          {item.name}
        </li>
      ))}
    </ul>
  );
}

// If you delete first item, the second item moves to position 0
// But React still sees key={0}, so it reuses the checkbox state
// Result: Checkbox states get mixed up!
```

### Q3: What's the difference between list items keeping their state with and without proper keys?

**Scenario:** A list with form inputs

```jsx
function FormItemList() {
  const [items] = useState([
    { id: 1, label: 'Name' },
    { id: 2, label: 'Email' },
    { id: 3, label: 'Phone' }
  ]);

  return items.map(item => <FormInput key={item.id} label={item.label} />);
}

function FormInput({ label }) {
  const [value, setValue] = useState('');
  
  return (
    <input
      value={value}
      onChange={(e) => setValue(e.target.value)}
      placeholder={label}
    />
  );
}

// With key={item.id}: Each input maintains its own state ✅
// With key={index}: When you filter/reorder, state gets mixed ❌
```

### Q4: Real Interview Scenario - Shopping Cart

```jsx
// Product array with ID
const [products, setProducts] = useState([
  { id: 101, name: 'Laptop', price: 50000, quantity: 0 },
  { id: 102, name: 'Phone', price: 30000, quantity: 0 },
  { id: 103, name: 'Tablet', price: 15000, quantity: 0 }
]);

function ShoppingCart() {
  // Remove product from cart
  const removeProduct = (id) => {
    setProducts(products.filter(p => p.id !== id));
  };

  // Reorder cart
  const moveToTop = (id) => {
    const item = products.find(p => p.id === id);
    const rest = products.filter(p => p.id !== id);
    setProducts([item, ...rest]);
  };

  return (
    <div>
      {products.map(product => (
        <CartItem
          key={product.id}  // ✅ Use ID, not index
          product={product}
          onRemove={() => removeProduct(product.id)}
          onMove={() => moveToTop(product.id)}
        />
      ))}
    </div>
  );
}

function CartItem({ product, onRemove, onMove }) {
  const [quantity, setQuantity] = useState(1);

  return (
    <div style={{ border: '1px solid #ccc', padding: '10px', margin: '10px' }}>
      <h3>{product.name}</h3>
      <p>Price: ${product.price}</p>
      <input
        type="number"
        value={quantity}
        onChange={(e) => setQuantity(e.target.value)}
        min="1"
      />
      <button onClick={onRemove}>Remove</button>
      <button onClick={onMove}>Move to Top</button>
    </div>
  );
}
```

**Interview Follow-up:** *What happens if you use index as key here?*
- When you remove first item, quantity input states get reassigned to wrong products
- The moved item loses its quantity state

### Q5: Dynamic List with Filtering

```jsx
function UserFilter() {
  const [users] = useState([
    { id: 1, name: 'Alice', role: 'Admin' },
    { id: 2, name: 'Bob', role: 'User' },
    { id: 3, name: 'Charlie', role: 'Admin' },
    { id: 4, name: 'David', role: 'User' }
  ]);

  const [selectedRole, setSelectedRole] = useState('');
  const [expanded, setExpanded] = useState({});

  // Filter users by role
  const filteredUsers = selectedRole 
    ? users.filter(u => u.role === selectedRole)
    : users;

  const toggleExpand = (id) => {
    setExpanded(prev => ({
      ...prev,
      [id]: !prev[id]
    }));
  };

  return (
    <div>
      <select value={selectedRole} onChange={(e) => setSelectedRole(e.target.value)}>
        <option value="">All</option>
        <option value="Admin">Admin</option>
        <option value="User">User</option>
      </select>

      <ul>
        {filteredUsers.map(user => (
          <li key={user.id} style={{ padding: '10px', border: '1px solid #ddd' }}>
            <span onClick={() => toggleExpand(user.id)} style={{ cursor: 'pointer' }}>
              {user.name} ({user.role}) {expanded[user.id] ? '▼' : '▶'}
            </span>
            {expanded[user.id] && (
              <div style={{ marginTop: '10px', paddingLeft: '20px' }}>
                <p>Details for {user.name}</p>
                <p>Role: {user.role}</p>
              </div>
            )}
          </li>
        ))}
      </ul>
    </div>
  );
}

// With key={user.id}: Expansion state stays with user ✅
// With key={index}: Expansion state breaks when filtered ❌
```

### Q6: Nested Lists Interview Question

```jsx
function CommentThread() {
  const [comments] = useState([
    {
      id: 1,
      author: 'Alice',
      text: 'Great post!',
      replies: [
        { id: 11, author: 'Bob', text: 'Thanks!' },
        { id: 12, author: 'Charlie', text: 'Agreed!' }
      ]
    },
    {
      id: 2,
      author: 'David',
      text: 'Interesting perspective',
      replies: [
        { id: 21, author: 'Eve', text: 'I disagree' }
      ]
    }
  ]);

  return (
    <div>
      {comments.map(comment => (
        <div key={comment.id} style={{ marginLeft: '0px', marginBottom: '20px' }}>
          <strong>{comment.author}:</strong> {comment.text}
          
          <div style={{ marginLeft: '20px' }}>
            {comment.replies.map(reply => (
              <div key={reply.id} style={{ marginBottom: '10px', paddingLeft: '10px' }}>
                <strong>{reply.author}:</strong> {reply.text}
              </div>
            ))}
          </div>
        </div>
      ))}
    </div>
  );
}

// Both levels need unique keys
// key={comment.id} for main comments
// key={reply.id} for nested replies
```

## Practice Question

### Exercise 1: Build a Todo List with Delete & Reorder

```jsx
function TodoApp() {
  const [todos, setTodos] = useState([
    { id: 1, text: 'Learn React', completed: false },
    { id: 2, text: 'Build Projects', completed: false }
  ]);

  const addTodo = () => {
    const newTodo = {
      id: Date.now(),
      text: 'New Todo',
      completed: false
    };
    setTodos([...todos, newTodo]);
  };

  const deleteTodo = (id) => {
    setTodos(todos.filter(todo => todo.id !== id));
  };

  const toggleTodo = (id) => {
    setTodos(todos.map(todo =>
      todo.id === id ? { ...todo, completed: !todo.completed } : todo
    ));
  };

  // Reorder: Move todo to top
  const moveToTop = (id) => {
    const todo = todos.find(t => t.id === id);
    const rest = todos.filter(t => t.id !== id);
    setTodos([todo, ...rest]);
  };

  return (
    <div>
      <button onClick={addTodo}>Add Todo</button>
      <ul>
        {todos.map(todo => (
          <li key={todo.id} style={{ textDecoration: todo.completed ? 'line-through' : 'none' }}>
            <input
              type="checkbox"
              checked={todo.completed}
              onChange={() => toggleTodo(todo.id)}
            />
            {todo.text}
            <button onClick={() => deleteTodo(todo.id)}>Delete</button>
            <button onClick={() => moveToTop(todo.id)}>Move Up</button>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

## Key Interview Takeaways

| Scenario | Key to Use | Why |
|----------|-----------|-----|
| Static list | index OK | No reordering/filtering |
| Dynamic with ID | **id** ✅ | Unique & stable |
| Filtered list | **id** ✅ | Index breaks filtering |
| Reordered list | **id** ✅ | Index breaks reordering |
| List with forms | **id** ✅ | Prevents state mixing |
| Nested lists | **Unique ID at each level** ✅ | Maintains hierarchy |

---

## Common Interview Mistakes

```jsx
// ❌ MISTAKE 1: Using array index
{items.map((item, index) => <Item key={index} />)}

// ❌ MISTAKE 2: Using non-unique values
{items.map(item => <Item key={item.name} />)}  // If duplicates exist!

// ❌ MISTAKE 3: Mixing key with dynamic lists
{items.map((item, index) => <Item key={Math.random()} />)}  // Creates new key every render!

// ✅ CORRECT: Use stable, unique ID
{items.map(item => <Item key={item.id} />)}
```

---

## Performance Impact

**With Proper Keys (key={id}):**
- React correctly updates only the changed DOM nodes
- Component state stays with the right item
- Better performance for large lists

**With Index Keys (key={index}):**
- React may recreate component instances unnecessarily
- State gets mixed up
- Performance degrades in dynamic lists
