## ⚠️ If You Get Stuck

### Problem Too Hard?
- Don't give up immediately
- Try these in order:
  1. Re-read problem (2 min)
  2. Plan on paper (5 min)
  3. Code step by step (don't try all at once)
  4. Console.log to debug (3 min)
  5. Google the error
  6. Try different approach
  7. THEN look at solution

### Stuck on Concept?
- Google: "React [concept] explained"
- Check React docs: react.dev
- Don't watch videos
- Read examples + code them

### Behind Schedule?
- Don't panic
- Add 1-2 extra days
- Revisit previous topics
- Speed comes with practice

## 🤔 Common Questions

**Q: What if I can't finish a problem in time?**  
A: That's OK! Stop anyway, check solution, code again next day. You'll be faster.

**Q: Should I watch tutorials on the side?**  
A: No. Coding is better learning. Only Google for specific errors.

**Q: What if I'm falling behind?**  
A: Add extra days. Day 11-12 can be review. No rush.

**Q: Should I memorize solutions?**  
A: No. Understand patterns. Code from memory. That's the goal.

**Q: When do I start Hard problems?**  
A: After Day 10 mastery check. If you code all 10 in < 2 hours correctly, you're ready.
